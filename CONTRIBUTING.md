Thank you for your interest in contributing! This repository is a student-maintained, documentation-style project meant for learning and sharing knowledge. Contributions are welcome and appreciated.

## How to contribute

1. **Open an issue** if you plan an addition or want to discuss scope and structure.
2. **Fork** the repository and make changes on a branch.
3. **Keep PRs focused** (one topic or fix per PR) and include a short description of why the change helps the project.
4. **Write clear commit messages** and add references or sources when appropriate.

## Coding style & content

- Prefer small, readable examples and add comments for clarity.
- For notebooks, ensure any required dependencies are noted in the file or a short `requirements.txt`.
- Add attribution when you include or adapt content from external sources.

## Pull request checklist

- [ ] Tests or runnable examples included (if applicable)
- [ ] Code and notebooks run without errors
- [ ] Documentation and comments added or updated
- [ ] Files added to appropriate folders with clear names
